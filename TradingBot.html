How did I make it?:

I use a GO api to track prices and data to send to Stocknet, it really helps having a live price feed always running from a VPS. The trading engine will always run flawless, and the api will always update, which leaves a never ending cycle of supply and demand gains without hiccups or lags

The reason I wrote this in C is because why not? its the fastest, and I have complete control over memory, pointers, threads, etc, etc allowing me to create not just a trading bot but a fully functional trading engine that never sleeps and never hiccups. A correctly written C program cant fail, they are designed to be for operations where failure is not an option. This bot was built from the metal up with 0 hand holding and 0 tutorials or guides.

all of the infomation I used to build this was form WINAPI docs, OANDAS api docs, Windows Internals book(2016), and online forums
